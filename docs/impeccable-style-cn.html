给 AI 讲清楚设计，不靠玄学

Impeccable 是把设计语言装进 AI 编程工具的一套技能包。

如果你已经会让 AI 写页面，但总觉得产出“能用却不耐看”，那这个项目解决的不是前端语法问题， 而是更前面的表达问题：你到底能不能用设计师听得懂、AI 也执行得动的话，描述你想要的界面。

As of 2026-03-28

一句话理解

它不是新的设计工具，也不是一个网页模板库，而是给 Cursor、Claude Code、Gemini CLI、Codex CLI 等 AI harness 增加“设计 fluency”的能力包。

截至 2026 年 3 月 28 日，官网展示的最新版本是 v1.6.0，发布日期为 2026 年 3 月 24 日。

01 / Quick Read

先把它看成三件事

1. 一套更会讲设计的话术

官网最核心的判断是：很多“AI 做出来像 AI”的页面，不是模型不会写代码，而是人不会提设计要求。

2. 一个底层技能 + 二十个命令

它在 Anthropic 原始的 `frontend-design` 技能基础上继续扩展，补上版式、颜色、审计、排版等更细的动作。

3. 给现有 AI 工具加“设计操作系统”

你不用换编辑器，也不用换模型，而是在现有工具里多出一组能直接调用的设计命令。

02 / Why It Exists

它在解决什么问题

常见但无效的提法

“帮我把页面做高级一点。” “要有科技感。” “再好看一点，不要那么普通。”

这种提示的问题是，听起来像方向，实际上没有设计结构。AI 往往会回到最常见的安全模板： 默认字体、千篇一律的卡片、泛紫渐变、平均分配的留白。

Impeccable 鼓励的提法

它试图把“设计师会怎么说”变成可以复用的命令和模式，比如版式节奏、信息层级、留白控制、排版修整、 视觉审计、风格强化、反模式识别。

换句话说，它想把“模糊审美”翻译成“可执行的设计动作”。

01
不是替你设计，而是帮你学会更精确地下设计指令。

这个项目的价值在于给非设计背景的人补一层设计词汇和设计判断框架。

02
不是只改样式，而是把“做得更好看”拆成一组具体动作。

例如审计、排版、整理布局、提纯视觉语言、增强特色，而不是一句万能的“polish”。

03
不是局限于单一平台，而是服务于多种 AI 开发工作流。

官网明确支持 Cursor、Claude Code、Gemini CLI、Codex CLI 等多种环境。

03 / Commands

最值得理解的是这组“命令语言”

设计命令示例

/polish

为什么这件事重要

当命令按设计任务来分工时，AI 的输出会更稳定，也更容易复盘到底是哪一步出了问题。

官网强调的变化

它不只提供技能，还提供反模式清单，帮助你避免“典型 AI 网页长相”。

你能得到什么

不是一套固定视觉模板，而是一套更接近设计工作流的提问方式和修正方式。

04 / Setup

怎么开始用

安装

官网给出的首选方式是一条统一命令，由工具自行识别你的 AI harness。

npx skills add pbakaus/impeccable

如果你已经在对应工具里使用 skill/command 体系，这个入口很直接。

更新

npx skills update

官网还提到可以先用 `npx skills check` 看更新内容；如果你用的是插件或手动下载 ZIP，也有对应更新路径。

适配工具

官网列出的支持范围覆盖多个常见 AI 编程环境：

Cursor Claude Code Gemini CLI Codex CLI VS Code Copilot Kiro OpenCode Trae Pi
05 / Fit

适合谁，不适合谁

更适合

  • 已经在用 AI 写前端，但对结果审美不满意的开发者
  • 没有系统学过设计，却想提升提示质量的产品经理或独立开发者
  • 希望团队能更一致地讨论界面问题的人

不必神化它

  • 它不能代替真实设计能力，只能帮你更接近设计思维
  • 如果你连基本的 AI 工具工作流都还不熟，官网也建议先学基础再回来
  • 它更像“设计指令层”，不是现成 UI 模板市场
